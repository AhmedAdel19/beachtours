Server Requirements

To get ManageMyBookings running on your server, you need to make sure your server matches the minimum requirements

1. Apache, nginx, or another compatible web server.
2. PHP >= 5.5.9 >> Higher
3. MySQL Database server
4. OpenSSL PHP Extension
5. Mbstring PHP Extension
6. Tokenizer PHP Extension
7. Module Re_write server
8. PHP_CURL Module Enable 

Preparation for MMB install

1. Download the app from ENVATO and unzip it.

2. In unzipped folder you will find folders : mmb – database – documentation

3. Create new database on your server or computer if you want to run it on localhost

4. You can use phpmyadmin or any other mysql database tools provided by your hosting company to import database file database/mmb.sql file into your new database

5. Now you are ready to upload all the files to your server and install the app

Install on your domain

You can install MMB app on a primary domain or a subdomain.

Install on your primary domain ( like http://www.yourdomain.com )

1. Put all MMB files and folders before www or public_html folder ( your account’s document root directory on a shared server is probably /home/username/public_html .. So all the files need to go / home/username/ )

2. After Upload is completed you will find a folder called Public. Now you can move all files and folders inside Public folder to www or public_html
3. that’s it 

 

Install on your subdomain

1. Create folder inside your www or public_html folder , then move all mmb files and folders to that folder

2. Create desired subdomain on your control panel and make sure you point your new subdomain location to your new subdomain folder example : public_html/myfoldername/public

3. that’s it

Setting Database Info

Go to main folder where you put all the files and open .env file using your favorite php editor. ( If you can’t find .env file, make sure on your Cpanel ( or any Hosting Panel you use ) you are able to see HIDDEN files.

DB_HOST=localhost

DB_DATABASE=mmb

DB_USERNAME=your database username

DB_PASSWORD=your database password

Site Settings

Allow Registration : Do you want new users to be able to register on your app? If yes then, check this box. New users or your staff will be able to register on your page. You don’t have to create every single user one by one!

Registration Activation: Do you let new users register? then choose how to activate their account!

Automatic Activation: As soon as they register they can enter the site! ( Not Recommended. You can get lots of unwanted users in your app )

Manual Activation: If you pick manual activation, then you have to activate all the accounts one by one! You have got full control ( isn’t that great ). Might be little bit time consuming but that’s ok! You got the power!

Email with activation link: In this case, new user will have an email with activation code and they will have to activate their new account!

Remember, you can remove any unwanted users from your system anytime!

Default Group Registration: Remember you can create as many groups of users as you like. Pick one group as a default registration group for your new users.

Captcha: Make things little bit difficult for your new users to register. Mixed up letters and alphabets to make registration little more difficult for bots ( Hopefully soon we will start using ReCaptcha )

Main Language: We made MMB multi language system. We know there are still some words that we missed and need to make it translatable. Please let us know if you find anything.

DateFormat: Simple. The format of the Date.. Pick one your site.

Email: What would you like to use for your emails to be delivered? PhpMailer or SWIFTMailer . If you choose SwiftMailer then make sure you follow all the instructions to configure it properly!

Metakey: Well it is just an old habit. Search engines don’t use them anylonger but it doesn’t hurt to some keywords for your website. By the way, this is for the frontend ! We haven’t implemented that yet. Hopefully soon

Meta Description : Description of your site. Pretty simple
Restricted IP Address: Any IP address you put in this field will NOT have an access to

your app.

Allowed IP Address: IP addresses written in this field will have an access to your app. Before you activate this field make sure that your IP address is in the list. 

 

Users & Groups

In MMB, you can add as many users and user groups as you like.
First Create different user groups ( NO:3 ) and give them different level of access! Remember “1” is LEVEL 1. That is SUPERADMIN that can do everything in the app!!

Want to create new user? First pick right user group and a unique username. Then fill up all the necessary fields. Pick the right status! If they are active, they can use the app with no issue. If inactive or Pending then they will have no access to your app.

Tours

We have created 2 different modules for tours and tour dates. You can define all the tour tours with with TOUR module. For each tour you create with Tour Module, you can add as many departure dates as you like and each departure date could have different features for the same tour. Tour capacity, price, tour code, tour guide and different departure type such as On Request, Daily, Set Dates..

First Create your tour. 

 

Fill up all the necessary fields and create your tour. After creating your tour go to Tour Page and create the itinerary. 

This is the SUB MENU. You can add new days to your itinerary or see al the departure dates or create beautiful PDF itineraries to send to your clients. 

This is the tour page you have. Add, edit , remove days. Create as many tours as you like.

After creating tour, you can create departure dates 

 

Tour Dates

Tour Date Module is where you can create departure dates for different tours.

First pick your tour category, ( this should be created before hand in under submodules ) then pick your tour under that tour category. Create your unique tour code. Next add tour start and end date. Group size is next. Pick if your tour is featured and/or Definite Departure.

Tour Features section is where you can define tour cost for single room, double room, triple room and a child.

You can change the status of tour anytime from Active to Inactive and even cancel the tour.

On the Tour Dates page, you can see the number of running tours, upcoming tours, past tours and cancelled tours

After creating your tour date , you can now start booking your tours.

This is the look of your tour departure date page where you can see all details about your departure.

When the tour is ready to go, you can get Booking Lists, Passport List and Emergency Details List as PDF File
